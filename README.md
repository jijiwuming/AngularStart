# 说明

该项目用于解释目录结构内容

# 本层目录结构

## e2e

### 端对端测试目录，主要是用于集成测试操作的，目前在项目中尚未开始使用

## node_modules

### 依赖包下载目录，在代码仓库中不存放

## src

### 主体的代码逻辑文件目录，我们主要在该目录下工作

> ## karma.conf.js
>
> ### karma 是单元测试的执行器，karma.conf.js 是 karma 的配置文件

> ## package.json
>
> ### 包管理器 npm 的配置文件，类似后端 pom.xml 文件，主要列出了该应用程序所使用的第三方依赖包
